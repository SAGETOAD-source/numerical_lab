Bisection Method

Root Finding Bracketing Guaranteed Convergence
🎯

The "Hot or Cold" Number Guessing Game

Remember the game where someone thinks of a number between 1–100 and you guess? The optimal strategy: always guess the exact middle. If it's too high, you eliminate the top half. Too low? Eliminate the bottom half. Each guess cuts your search space in half!

Bisection does exactly this for equations. If f(1) is negative and f(2) is positive, there's a root somewhere between 1 and 2. Check the midpoint 1.5. Negative? Root is in [1.5, 2]. Positive? Root is in [1, 1.5]. Repeat until the interval is tiny — that's your root!

Mathematical Definition

Given a continuous function f(x) where f(a)·f(b) < 0 (sign change guarantees a root by IVT):

c = (a + b) / 2
If f(a)·f(c) < 0 → new interval is [a, c]
If f(b)·f(c) < 0 → new interval is [c, b]
Max Error ≤ (b − a) / 2ⁿ

The error halves each iteration. After n steps, the error is at most (b−a)/2ⁿ. This is called linear convergence.

f(a) · f(b) < 0

Checks that the function has opposite signs at both ends — this is the Intermediate Value Theorem guarantee. Without it, the method is invalid.

c = (a+b)/2

The new guess is always the exact middle of the current interval — maximally conservative but guarantees progress every iteration.

Error = |b−a|/2

Each iteration halves the interval, so error halves too. It's slow but perfectly predictable — you know exactly how many steps you need.

When does it stop?

When |f(c)| is smaller than a tolerance (found the root) OR when the interval [a,b] is smaller than our precision requirement.

// Bisection Method — finds root of f(x) in [a, b]
// Prerequisite: f(a) and f(b) must have opposite signs

function bisection(f, a, b, maxIter = 20, tol = 1e-8) {
  // Verify the bracketing condition (Intermediate Value Theorem)
  if (f(a) * f(b) >= 0) {
    throw new Error("f(a) and f(b) must have opposite signs!");
  }

  let results = [];

  for (let i = 0; i < maxIter; i++) {
    const c  = (a + b) / 2;   // Midpoint = our current best guess
    const fc = f(c);              // Evaluate the function at midpoint
    const err = (b - a) / 2;    // Maximum possible error

    results.push({ step: i+1, a, b, c, fc, err });

    if (Math.abs(fc) < tol || err < tol) break; // Converged!

    // The root is in whichever half has the sign change
    if (f(a) * fc < 0) b = c;  // Root in left half
    else                a = c;  // Root in right half
  }

  return results;
}

// Usage
const f = x => x**3 - x - 2;
const steps = bisection(f, 1, 2);
console.log("Root ≈", steps.at(-1).c); // ≈ 1.5213797

False Position

Root FindingRegula FalsiWeighted Guess
📐

Drawing a Smarter Shortcut

Bisection always picks the exact middle of an interval — ignoring information about how steep the curve is on each side. False Position is smarter: it draws a straight line connecting the two endpoint values and asks "where does this line hit zero?"

If one endpoint has a function value of 100 and the other has −1, the root is probably very close to the second endpoint — not in the middle! False Position naturally leans toward the correct side, often converging much faster.

Mathematical Definition

The x-intercept of the secant line through (a, f(a)) and (b, f(b)):

c = b − f(b) · (b − a) / (f(b) − f(a))
This is the line y = f(a) + [f(b)−f(a)]/(b−a) · (x−a) solved for y=0
vs Bisection

Bisection picks arithmetic midpoint. False Position picks weighted midpoint — weighted by function values. When the function is steep on one side, it leans toward that side.

The "Stagnation" Problem

False Position can get stuck: one endpoint may never update (the root keeps landing on the same side). The modified version (Illinois method) fixes this.

function falsePosition(f, a, b, maxIter = 20) {
  if (f(a) * f(b) >= 0) throw new Error("Need opposite signs!");
  let prev = a, results = [];

  for (let i = 0; i < maxIter; i++) {
    const fa = f(a), fb = f(b);

    // x-intercept of line through (a,fa) and (b,fb)
    const c  = b - fb * (b - a) / (fb - fa);
    const fc = f(c);
    const err = Math.abs(c - prev);
    results.push({ step:i+1, a, b, c, fc, err });
    prev = c;

    if (Math.abs(fc) < 1e-8 || err < 1e-8) break;
    if (f(a) * fc < 0) b = c; else a = c;
  }
  return results;
}

Newton-Raphson

Root FindingOpen MethodQuadratic Convergence
🏹

Following the Slope to Zero

Imagine standing on a hill in fog. You can feel the slope under your feet. The steeper the slope, the longer your stride. You draw an imaginary straight tangent line through the point where you're standing, then follow it all the way to where it would hit flat ground — and you jump there.

That's Newton-Raphson: at any point, draw the tangent (using the derivative), see where it crosses zero — that's your next guess. The genius is that errors square each iteration. If error is 0.01, next step gives 0.0001, then 0.00000001. Incredibly fast!

Mathematical Definition

x_{n+1} = xₙ − f(xₙ) / f′(xₙ)
f′(x) is the derivative of f(x). The tangent line at (xₙ, f(xₙ)) crosses zero at x_{n+1}.
Error ≈ C · (eₙ)²  — Quadratic Convergence
Quadratic convergence

The number of correct decimal places roughly doubles each iteration! Bisection adds ~0.3 decimal places per step. Newton-Raphson doubles them.

The danger zone

If f′(xₙ) ≈ 0 (near a local max or min), the tangent is nearly flat and shoots the next guess far away. The method can diverge dramatically.

Requires derivative

Unlike bisection, you must know f′(x) analytically (or numerically). This is a significant requirement in practice.

No bracket needed

Newton-Raphson is an "open" method — you only need one starting point x₀, not an interval. But it doesn't guarantee convergence.

function newtonRaphson(f, df, x0, maxIter = 20, tol = 1e-10) {
  let x = x0, results = [];

  for (let i = 0; i < maxIter; i++) {
    const fx  = f(x);             // Current function value
    const dfx = df(x);            // Slope of tangent at current x

    if (Math.abs(dfx) < 1e-12)   // Guard: flat tangent = danger!
      throw new Error("Derivative near zero");

    // Follow tangent line to x-axis: x_new = x - f(x)/f'(x)
    const xNew = x - fx / dfx;
    const err  = Math.abs(xNew - x);
    results.push({ step:i+1, x, fx, dfx, xNew, err });
    x = xNew;

    if (err < tol) break; // Converged!
  }
  return results;
}

Lagrange Interpolation

InterpolationPolynomial Fitting
🌉

Building a Bridge Between Known Points

You measure the temperature at 8am = 20°C, noon = 32°C, and 6pm = 28°C. What was it at 3pm? You don't know — but you can build a smooth curve that passes exactly through all your known points, then read off any value in between.

Lagrange builds this bridge using a clever trick: for each known point, it creates a special "basis" polynomial that equals 1 at that point and 0 at all others. Then it blends them all together with the measured values as weights.

Mathematical Definition

P(x) = Σᵢ yᵢ · Lᵢ(x)
Lᵢ(x) = ∏_{j≠i} (x − xⱼ) / (xᵢ − xⱼ)
Each Lᵢ(xᵢ) = 1 and Lᵢ(xⱼ) = 0 for j≠i — so P exactly passes through all data points

Input Parameters

Interpolated Curve

Basis Polynomials
ixᵢyᵢLᵢ(x)yᵢ · Lᵢ(x)
JavaScript
function lagrange(xs, ys, x) {
  const n = xs.length;
  let result = 0;

  for (let i = 0; i < n; i++) {
    // Build the i-th basis polynomial L_i(x)
    let Li = 1;
    for (let j = 0; j < n; j++) {
      if (j !== i) {
        // Each factor: (x-xj)/(xi-xj)
        // Makes Li = 0 at all xj, and Li = 1 at xi
        Li *= (x - xs[j]) / (xs[i] - xs[j]);
      }
    }
    result += ys[i] * Li; // Weight by y value
  }
  return result;
}

Euler's Method

ODE SolverInitial Value ProblemFirst Order
🚶

Walking in Fog with a Compass

You're navigating through thick fog and must follow a curving path. You have a compass that tells you the exact direction to walk right now, but you can only check it periodically. So you walk in a straight line for a short distance, stop, check the compass again, walk in that new direction — and so on.

Each "straight line segment" is one Euler step. The smaller your steps, the more accurately you follow the true curved path. But more steps = more computation. This is the fundamental tradeoff of numerical ODE solving.

Mathematical Definition

Solves dy/dx = f(x,y) with y(x₀) = y₀:

y_{n+1} = yₙ + h · f(xₙ, yₙ)
x_{n+1} = xₙ + h
h = step size. Smaller h → more accurate but slower. Error ∝ h (first order).

Solution Curve

function euler(f, x0, y0, xEnd, h) {
  let x = x0, y = y0;
  const res = [{ step:0, x, y }];
  const N = Math.round((xEnd - x0) / h);

  for (let i = 0; i < N; i++) {
    const slope = f(x, y); // Direction to walk (the derivative)
    y = y + h * slope;    // Take one step in that direction
    x = +(x + h).toFixed(10); // Advance x
    res.push({ step:i+1, x, y, slope });
  }
  return res;
}
// Example: dy/dx = x+y, y(0)=1 → exact: y = 2eˣ − x − 1
const steps = euler((x,y) => x+y, 0, 1, 2, 0.1);

Runge-Kutta (RK4)

ODE Solver4th OrderGold Standard
🧭

Four Compass Readings Per Step

Euler checks the compass only at the start of each step, then walks blind. Imagine instead checking the compass four times per step: at the start, twice at the midpoint (from two different directions), and once at the end — then take a carefully weighted average of all four directions.

That's RK4. The brilliant weights 1, 2, 2, 1 come from Simpson's rule — it turns out that sampling the slope at midpoints twice is more valuable than once at the endpoints. The result? Error shrinks as h⁴ instead of h — same step size, massively better accuracy.

Four Slope Estimates

k₁ = h·f(xₙ, yₙ)        ← slope at start
k₂ = h·f(xₙ+h/2, yₙ+k₁/2) ← slope at midpoint 1
k₃ = h·f(xₙ+h/2, yₙ+k₂/2) ← slope at midpoint 2
k₄ = h·f(xₙ+h, yₙ+k₃)   ← slope at end
y_{n+1} = yₙ + (k₁ + 2k₂ + 2k₃ + k₄) / 6
Midpoints count double — they're more representative of the step's true slope

JavaScript
function rk4(f, x0, y0, xEnd, h) {
  let x = x0, y = y0;
  const res = [{ step:0, x, y }];
  const N = Math.round((xEnd - x0) / h);

  for (let i = 0; i < N; i++) {
    // Four slope samples at different points in the interval
    const k1 = h * f(x,         y          );
    const k2 = h * f(x + h/2,   y + k1/2   );
    const k3 = h * f(x + h/2,   y + k2/2   );
    const k4 = h * f(x + h,     y + k3      );

    // Weighted average: midpoints (k2,k3) count twice
    y += (k1 + 2*k2 + 2*k3 + k4) / 6;
    x = +(x + h).toFixed(10);
    res.push({ step:i+1, x, y, k1, k2, k3, k4 });
  }
  return res;
}

Trapezoidal Rule

Numerical IntegrationArea ApproximationLinear
🏔

Measuring a Hilly Field with Fences

You want to measure the area of an irregularly shaped field. Divide it into vertical strips. Each strip's top edge follows the curve, but you approximate it as a straight line connecting the left and right heights — making it a trapezoid.

Area of one trapezoid = (left height + right height) / 2 × width. Sum all strips. More strips → closer to the true curved area. The error shrinks proportional to h² — doubling the strips quarters the error!

Mathematical Definition

∫f(x)dx ≈ h/2 · [f(x₀) + 2f(x₁) + 2f(x₂) + ··· + 2f(x_{n-1}) + f(xₙ)]
h = (b − a) / n
Pattern: 1, 2, 2, 2, ..., 2, 1 — endpoints get weight 1, interior points get weight 2
Error ∝ h² — second order accuracy

JavaScript
function trapezoidal(f, a, b, n) {
  const h = (b - a) / n;            // Width of each strip
  let sum = f(a) + f(b);             // Endpoints: coefficient 1

  for (let i = 1; i < n; i++) {
    // Interior points: coefficient 2 (shared by two trapezoids)
    sum += 2 * f(a + i * h);
  }

  return (h / 2) * sum;  // Area = (h/2) * total sum
}

// ∫sin(x)dx from 0 to π = 2 (exact)
console.log(trapezoidal(Math.sin, 0, Math.PI, 100));

Simpson's 1/3 Rule

Numerical IntegrationParabolic Arcs4th Order
🎪

Arching Over Instead of Slicing Straight

The trapezoidal rule connects points with straight lines. But real curves aren't straight! Simpson's rule says: fit a parabola through every three consecutive points and integrate that curved shape exactly. A tent with a curved ceiling is far closer to the actual shape than one with flat panels.

The magic: for polynomials up to degree 3, Simpson's gives the exact answer even with just a few strips. Error shrinks as h⁴ — twice the order of trapezoidal. That's an enormous practical difference.

Mathematical Definition

∫f(x)dx ≈ h/3 · [f(x₀) + 4f(x₁) + 2f(x₂) + 4f(x₃) + ··· + 4f(x_{n-1}) + f(xₙ)]
Pattern: 1, 4, 2, 4, 2, ..., 4, 1 — n must be even!
Error ∝ h⁴ — fourth order accuracy

Input Parameters

Parabolic Arcs

JavaScript
function simpsons(f, a, b, n) {
  if (n % 2 !== 0) n++;         // n must be even
  const h = (b - a) / n;
  let sum = f(a) + f(b);         // Endpoints: coefficient 1

  for (let i = 1; i < n; i++) {
    const x = a + i * h;
    // Odd index → coefficient 4, Even → coefficient 2
    sum += (i % 2 === 0 ? 2 : 4) * f(x);
  }

  return (h / 3) * sum;   // (h/3) — Simpson's formula
}
// Exact for polynomials up to degree 3!

Gauss-Seidel Method

Linear SystemsIterative SolverAx = b
🔄

Friends Taking Turns to Balance Accounts

Three friends share complicated finances. Each person owes money to the others in different amounts. To settle everything, they take turns: person 1 recalculates what they owe based on the current best guesses of everyone else, then updates their balance. Person 2 does the same, immediately using person 1's updated value. Round and round — balances stabilize as everyone converges.

Gauss-Seidel does exactly this: solve each equation for one variable, immediately use the latest value. Unlike Jacobi (which waits for a full round), using fresh values immediately makes it converge about twice as fast.

Mathematical Definition

For system Ax = b, isolate each xᵢ and iterate:

x_i^(k+1) = (bᵢ − Σ_{j<i} aᵢⱼ·xⱼ^(k+1) − Σ_{j>i} aᵢⱼ·xⱼ^(k)) / aᵢᵢ
j<i terms use NEW values (already updated this round)
j>i terms use OLD values (not yet updated this round)
Converges when |aᵢᵢ| > Σ_{j≠i}|aᵢⱼ| (diagonally dominant)

3×3 System — Enter Matrix [A | b]

| = x₁ row
| = x₂ row
| = x₃ row

function gaussSeidel(A, b, maxIter = 25, tol = 1e-8) {
  const n = A.length;
  let x = new Array(n).fill(0); // Start with all zeros
  const results = [];

  for (let iter = 0; iter < maxIter; iter++) {
    const xOld = [...x]; // Snapshot for error calculation

    for (let i = 0; i < n; i++) {
      let sum = 0;
      for (let j = 0; j < n; j++) {
        if (j !== i) sum += A[i][j] * x[j]; // Uses newest x!
      }
      // Isolate x[i]: a_ii * x_i = b_i - sum
      x[i] = (b[i] - sum) / A[i][i];
    }

    const err = Math.max(...x.map((xi,i) => Math.abs(xi - xOld[i])));
    results.push({ iter:iter+1, x:[...x], err });
    if (err < tol) break; // Converged!
  }
  return results;
}
